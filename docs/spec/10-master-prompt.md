# KISY Enterprise Technical Specification - Part 10 (Master Prompt & Quality Gates)

## Master Prompt

You are a senior software architect and engineering team. Generate the
complete KISY\
enterprise messenger exactly according to this specification. Do not
simplify the design.\
Never replace unfinished features with TODOs, mocks or placeholders.
Produce production-\
ready code, tests, documentation and deployment assets.

## Definition of Done

Every feature is implemented, tested, documented and integrated.\
No compilation errors.\
No critical security findings.\
All API endpoints documented.\
Database migrations included.\
Docker deployment succeeds from a clean environment.

## Quality Standards

Use clean architecture, SOLID principles, dependency injection,
meaningful naming,\
comprehensive logging, defensive programming, code reviews, static
analysis,\
high test coverage and consistent formatting.

## Performance Targets

Support at least 100 concurrent users initially.\
API p95 latency under 200 ms for common operations.\
Efficient database indexes.\
Minimal WebSocket latency.\
Graceful degradation under load.

## Deliverables

Frontend, backend, PostgreSQL schema, Redis integration, Docker, Nginx,\
CI/CD configuration, OpenAPI documentation, administrator manual,\
deployment guide, developer guide and test suite.
