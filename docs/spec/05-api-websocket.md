# KISY Enterprise Technical Specification - Part 5 (REST API & WebSocket)

## API Principles

REST API under /api/v1. JSON only. UTC timestamps in ISO-8601. UUID
identifiers.\
Consistent error format: code, message, details, requestId. Versioning
required.\
Every endpoint protected by RBAC unless explicitly public.

## Voice Call Signaling (1:1 audio)

Audio calls use WebRTC; media flows peer-to-peer. Only the signaling
(SDP offer/answer and ICE candidates) travels over this WebSocket, and the
server merely relays each frame to the other participant after authorizing
that the sender and target share the direct chat (the same visibility rule as
messaging — reusing the chat participant check). Each call has a `callId`
(UUID generated by the initiator); the server validates that the sender of any
`call.*` frame is a participant of that call.

Client→Server:
- `call.invite` — `{ callId, toUserId, chatId, sdp }` (SDP offer)
- `call.answer` — `{ callId, sdp }` (SDP answer)
- `call.ice` — `{ callId, candidate }` (trickle ICE candidate)
- `call.reject` — `{ callId }` (callee declines)
- `call.cancel` — `{ callId }` (caller aborts before answer)
- `call.hangup` — `{ callId }` (either party ends an answered call)

Server→Client:
- `call.incoming` — `{ callId, from: { id, displayName, avatarUrl }, chatId, sdp }`
- `call.answered` — `{ callId, sdp }`
- `call.ice` — `{ callId, fromUserId, candidate }`
- `call.rejected` / `call.canceled` / `call.busy` / `call.timeout` — `{ callId }`
- `call.ended` — `{ callId, reason }`

Server rules: a call to a user already on a call yields `call.busy`; an
unanswered call rings for 45 s then `call.timeout` (recorded as *missed*);
invites are rate-limited per caller; unauthorized or malformed signaling yields
a generic `error` frame (internal details are never leaked). Every call is
written to `call_logs` and lifecycle transitions are audit-logged. STUN/TURN
configuration is fetched over REST (`GET /calls/ice-config`); TURN credentials
are short-lived and derived from the coturn shared secret server-side.
